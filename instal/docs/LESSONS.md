## Что ломалось и как чинили

1. **Pump с nest_asyncio — обязателен.** Без него `Kernel.do_one_iteration()` (async корутина) из синхронного кода не вызывается — RuntimeWarning. Решение: `nest_asyncio.apply()` + `loop.run_until_complete(res)`.

2. **При рефакторинге — не выбрасывать сложные механизмы.** `_make_kernel_pump()` был в рабочем `start.py`, выброшен при разбивке на модули → кнопки сломались.

3. **Порядок проверок в `_wait_for_port()`:** сначала порт (socket), потом `comfy_proc.poll()`. Иначе race condition при restart → false positive SIGKILL.

4. **SDPA (torch default)** — работает на T4, 720p 2 прохода без OOM. Медленно на GGUF.

5. **SageAttention** — НЕ работает с GGUF. Только для диффузионных моделей.

6. **CUDA 13 (cu130)** — наша версия, не cu124.

7. **llamacpp_gguf_cuda wheel** — не под Python 3.12 (wheel под 3.11). И не для ComfyUI (часть WanGP).

### Scroll лога: HTML с buffer + flusher > Output.append_stdout

**Проблема:** Output.append_stdout() триггерит авто-скролл frontend'а на каждый вызов.
Даже с батчингом раз в 0.5с — скролл прыгал, пользователь не мог читать старые логи.

**Решение:** widgets.HTML с `<pre>` внутри, overflow:auto на контейнере. Браузерное
CSS scroll anchoring (Chrome 77+, стандарт) само решает, когда скроллить:
- Если пользователь внизу — новые строки смещают старые, скролл остаётся внизу
- Если читает выше — скролл не сдвигается

**Эволюция (4 попытки):**
1. HTML per-line → скролл наверх
2. Output per-line → скролл вниз (iopub flood)
3. Output + 0.5s buffer → скролл вниз раз в 0.5с
4. HTML + 0.5s buffer → **scroll anchoring, без дёрганья** ✅

**Вывод:** Не бороться с авто-скроллом Output через батчинг — это never работает
на Jupyter frontend'ах (нет API для отключения). HTML-виджет даёт scroll anchoring
бесплатно, без единой строчки JS.
